# RAG チャットボット要件定義書（修正版）

## 1. プロジェクト概要

### 1.1 目的

- 社内文書を対象としたセマンティック検索チャットボットの構築
- RAG の基本実装から最適化までを学習
- 実際にデプロイ可能な状態まで完成させる

### 1.2 スコープ

- 社内文書（PDF, Word, テキスト）の検索機能
- 自然言語での質問応答
- パフォーマンスチューニング機能の実装
- Web UI の提供
- クラウドへのデプロイ

### 1.3 技術スタック

- フレームワーク：LangChain
- ベクトル DB：ChromaDB
- UI：Streamlit
- LLM：Google Gemini 2.5 Flash
- Embedding モデル：Google text-embedding-004
- API：Google AI Studio
- デプロイ：Streamlit Community Cloud

## 3. 機能要件

### 3.1 ドキュメント管理機能

#### 3.1.1 ドキュメント読み込み

- 対応フォーマット：PDF、DOCX、TXT、MD
- 複数ファイルの一括読み込み
- エラーハンドリング（破損ファイル、非対応フォーマット）

#### 3.1.2 前処理

- テキスト抽出
- 文字コード正規化
- 不要な空白・改行の除去

### 3.2 インデックス作成機能

#### 3.2.1 テキスト分割（チャンキング）

- 分割方法：RecursiveCharacterTextSplitter 使用
- パラメータ：
  - chunk_size：設定可能（デフォルト 500）
  - chunk_overlap：設定可能（デフォルト 50）
  - separators：["\n\n", "\n", "。", "、", " ", ""]

#### 3.2.2 Embedding 生成

- モデル：Google text-embedding-004
- バッチ処理対応
- エラーリトライ機能
- レート制限対応

#### 3.2.3 ベクトル DB 保存

- ChromaDB にコレクション作成
- メタデータ付与：
  - source：ファイル名
  - page：ページ番号（該当する場合）
  - chunk_id：チャンク ID
  - created_at：作成日時

### 3.3 検索・回答生成機能

#### 3.3.1 検索機能

- セマンティック検索
- 取得件数：設定可能（デフォルト 5 件）
- 類似度スコア閾値：設定可能

#### 3.3.2 回答生成

- LLM：Google Gemini 2.5 Flash
- プロンプトテンプレート：カスタマイズ可能
- ストリーミング出力対応

#### 3.3.3 ソース表示

- 参照したドキュメントの表示
- ページ番号の表示
- 類似度スコアの表示

### 3.4 UI 機能

#### 3.4.1 メイン画面

- チャットインターフェース
- 質問入力欄
- 回答表示エリア
- ソース情報の折りたたみ表示

#### 3.4.2 サイドバー

- ドキュメントアップロード
- インデックス作成ボタン
- パラメータ調整パネル：
  - chunk_size 調整スライダー
  - chunk_overlap 調整スライダー
  - top_k（取得件数）調整
  - temperature 調整
- パフォーマンス表示エリア

#### 3.4.3 管理画面

- インデックス済みドキュメント一覧
- ドキュメント削除機能
- インデックス再構築機能

### 3.5 パフォーマンスチューニング機能

#### 3.5.1 チャンキング最適化

- チャンクサイズの動的変更
- A/B テスト機能：
  - 複数のチャンクサイズで並列実行
  - 結果の比較表示
- 評価指標：
  - 検索精度（手動評価用）
  - レスポンス時間
  - 取得チャンク数

#### 3.5.2 パフォーマンス計測

- 検索時間の計測
- Embedding 生成時間の計測
- LLM 応答時間の計測
- 総処理時間の表示

#### 3.5.3 最適化レポート

- チャンクサイズごとの性能比較
- 推奨設定の提案
- グラフ表示（Streamlit チャート）

## 4. 非機能要件

### 4.1 パフォーマンス

- 検索応答時間：5 秒以内（目標）
- インデックス作成：ドキュメント数に応じて数分程度
- 同時接続：5-10 ユーザー程度を想定

### 4.2 可用性

- アップタイム：開発環境のため制約なし
- データ永続化：ローカルファイルシステム使用

### 4.3 セキュリティ

- API キーの環境変数管理
- .env ファイルの Git 除外
- 基本的な入力バリデーション

### 4.4 スケーラビリティ

- 小規模データ（数百〜数千ドキュメント）を想定
- 将来的な拡張を考慮した設計

## 5. データ仕様

### 5.1 ドキュメントメタデータ

```python
{
    "source": str,          # ファイル名
    "page": int,            # ページ番号（該当する場合）
    "chunk_id": str,        # 一意のチャンクID
    "created_at": str,      # ISO 8601形式の日時
    "chunk_size": int,      # 使用したチャンクサイズ
    "chunk_overlap": int    # 使用したオーバーラップ
}
```

### 5.2 検索結果形式

```python
{
    "answer": str,              # 生成された回答
    "sources": [
        {
            "content": str,     # チャンクの内容
            "metadata": dict,   # メタデータ
            "score": float      # 類似度スコア
        }
    ],
    "performance": {
        "search_time": float,   # 検索時間（秒）
        "generation_time": float,  # 生成時間（秒）
        "total_time": float     # 総時間（秒）
    }
}
```

## 8. Google AI Studio API 仕様

### 8.1 認証

**API キーの取得**

1. Google AI Studio（https://aistudio.google.com/）にアクセス
2. 「Get API Key」をクリック
3. API キーを生成
4. .env ファイルに保存

**環境変数**

```
GOOGLE_API_KEY=your-api-key-here
```

### 8.2 LLM API（Gemini）

**LangChain での使用**

```python
from langchain_google_genai import ChatGoogleGenerativeAI

llm = ChatGoogleGenerativeAI(
    model="gemini-2.5-flash",
    google_api_key=os.getenv("GOOGLE_API_KEY"),
    temperature=0,
    max_output_tokens=2048
)
```

**主な特徴**

- 無料枠：1 日あたり 1,500 リクエスト
- レート制限：1 分あたり 15 リクエスト
- コンテキストウィンドウ：最大 100 万トークン（Gemini 2.5 Flash）
- 出力トークン上限：8,192 トークン
- ストリーミング対応

### 8.3 Embedding API

**LangChain での使用**

```python
from langchain_google_genai import GoogleGenerativeAIEmbeddings

embeddings = GoogleGenerativeAIEmbeddings(
    model="models/text-embedding-004",
    google_api_key=os.getenv("GOOGLE_API_KEY"),
    task_type="retrieval_document"
)
```

**task_type の種類**

- retrieval_document：ドキュメントの Embedding
- retrieval_query：クエリの Embedding
- semantic_similarity：類似度計算用
- classification：分類用

**主な特徴**

- 無料枠：1 日あたり 1,500 リクエスト
- 次元数：768 次元（デフォルト）
- バッチサイズ：最大 100 件
- コスト：無料

### 8.4 レート制限対応

**リトライ戦略**

```python
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=4, max=10)
)
def call_gemini_api():
    # API呼び出し
    pass
```

**バッチ処理**

- Embedding は 100 件ずつバッチ処理
- リクエスト間に適切な待機時間を設定

## 9. エラーハンドリング

### 9.1 想定されるエラー

- ファイル読み込みエラー

- レート制限エラー（429）
- API キー認証エラー（401）
- ChromaDB 接続エラー
- メモリ不足エラー

### 9.2 エラー対応

**API 関連エラー**

```python
try:
    response = llm.invoke(prompt)
except Exception as e:
    if "429" in str(e):
        st.error("レート制限に達しました。しばらく待ってから再試行してください。")
    elif "401" in str(e):
        st.error("APIキーが無効です。設定を確認してください。")
    else:
        st.error(f"エラーが発生しました: {str(e)}")
```

**一般的なエラー対応**

- ユーザーフレンドリーなエラーメッセージ表示
- リトライ機能（API 呼び出し）
- ログ出力（デバッグ用）
- フォールバック処理

## 10. デプロイ要件

### 10.1 デプロイ先

**Streamlit Community Cloud（推奨）**

- 理由：
  - Streamlit アプリに特化
  - 無料枠あり
  - GitHub と連携が容易
  - 設定が簡単
- 注意：
  - ChromaDB のデータ永続化が制限される
  - 再起動時にデータが消える可能性

### 10.2 環境変数

**Streamlit Community Cloud の secrets 設定**

```toml
# .streamlit/secrets.toml（ローカル開発用）
GOOGLE_API_KEY = "your-api-key-here"
```

**Web UI での設定**

1. Streamlit Community Cloud ダッシュボード
2. アプリ設定を開く
3. 「Secrets」セクションに上記と同じ内容を入力

### 10.3 デプロイ手順

1. GitHub リポジトリ作成
2. コードをプッシュ
3. Streamlit Community Cloud でアプリ作成
4. 環境変数（GOOGLE_API_KEY）を設定
5. デプロイ実行

### 10.4 制限事項

**Streamlit Community Cloud**

- ファイルサイズ制限：200MB
- メモリ制限：1GB
- 同時接続数：制限あり
- スリープモード：一定時間アクセスがないとスリープ

**Google AI Studio API**

- 無料枠：1 日あたり 1,500 リクエスト
- レート制限：1 分あたり 15 リクエスト
- 商用利用には制限がある場合がある
